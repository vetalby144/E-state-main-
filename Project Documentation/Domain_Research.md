# Початкове дослідження предметної області

## Мета проєкту
Створення вебсайту/системи для компанії **E-state**, яка займається продажем, орендою та інвестиціями у нерухомість.  
Система повинна дозволяти користувачам:

- переглядати об’єкти,  
- отримувати детальну інформацію про них,  
- фільтрувати за параметрами,  
- зв’язуватися з агентами.

---

## 1. Аналіз існуючих рішень
На ринку вже існують подібні системи:

- **DOM.RIA**, **OLX Нерухомість**, **Lun.ua**, **Flatfy**, **Address.ua**  
- Зарубіжні аналоги: **Zillow**, **Realtor.com**, **Redfin**

**Основні функції, які вони мають:**

- Пошук об’єктів за фільтрами (ціна, локація, тип, площа тощо)  
- Карта з розташуванням об’єктів  
- Галереї фотографій  
- Контактна форма або чат з агентом  
- Кабінет користувача/агента (особисті оголошення, статистика)  
- Система рейтингу або коментарів

---

## 2. Потенційний функціонал системи E-state
Для реалізації проєкту можуть знадобитися такі модулі:

- **Каталог об’єктів** — перегляд списку нерухомості  
- **Пошук і фільтри** — за ціною, типом, локацією, площею  
- **Сторінка об’єкта** — фото, опис, характеристики, контакт агента  
- **Адмінпанель** (на наступних етапах) — додавання, редагування, видалення об’єктів  
- **Контактна форма** — для запиту консультації  
- **Адаптивний інтерфейс** — коректне відображення на смартфонах  
- *(Опціонально)* Вхід/реєстрація користувачів

---

## 3. Що потрібно для реалізації
- **Frontend:** HTML, CSS, JavaScript, можливо React/Vue  
- **Backend:** Node.js (Express) або Python (Flask/Django)  
- **База даних:** MongoDB / MySQL (для зберігання об’єктів)  
- **Хостинг:** Render, Vercel, або власний сервер  
- **API:** для передачі даних між frontend і backend
